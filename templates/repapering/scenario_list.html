Repapering Architecture
Config File Structure

The system synchronizes all UI mappings into repapering_config.json in the project root. This JSON acts as the source of truth for the document evaluation backend.

Rules & Conditions

The Rules / Special Conditions field contains logical constraints (e.g., ClientType == 'NP') and is read-only in the interface to prevent accidental breakage of technical requirements. It is populated by the backend/config file.

Audit & Versioning

Changes in the UI are tracked via HistoricalRecords. Additionally, repapering_config.json should be version-controlled via Git to maintain history of rule updates.
